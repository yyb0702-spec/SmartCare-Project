## 1. 제품 개요
DoctorPet은 반려동물 증상을 입력받아 AI가 방문 필요성과 필요 진료역량을 안내하고, 근처 동물병원 검색부터 예약·진료·후불 결제까지 연결하는 서비스다.
```plain text
증상 입력
→ AI 진료역량 구조화·검색 Tool 호출
→ 병원 선택
→ 예약 요청
→ 병원 승인
→ 체크인·진료
→ 후불 결제
```
## 2. 해결하려는 문제
### 보호자
- 증상에 맞는 진료역량과 병원을 찾기 어렵다.
- 병원의 진료 분야·검사·시설·장비를 비교하기 어렵다.
- 검색, 예약, 진료, 결제가 분리되어 있다.
### 병원
- 실제 일정과 진료 가능 여부를 반영하여 예약을 승인해야 한다.
- 예약자의 전 병원 통합 취소·노쇼 이력을 확인할 필요가 있다.
- 진료 완료 후 결제 실패와 오프라인 수납을 관리해야 한다.
## 3. 사용자
| 사용자 | 주요 기능 |
| --- | --- |
| 보호자 | 반려동물 등록, 병원 검색, AI 상담, 예약, 결제 |
| 병원 스태프 | 예약 승인·거절, 체크인, 진료 상태, 진료비 청구·정산 |
AI 상담은 비로그인 사용자도 임시 정보로 이용할 수 있다. MVP에서 병원 전용 프론트 화면은 제외하며 병원 기능은 API로 검증한다.

| 도메인 | 기능 |

## 5. 핵심 요구사항
- 제휴 병원은 공공데이터와 더미 제휴 데이터를 지자체 코드+관리번호 복합 키로 매핑하며 복합 UNIQUE를 적용한다.
- 비제휴 병원은 참고 정보만 제공하고 예약을 허용하지 않는다.
- 슬롯과 예약 이력은 1:N이며, 동일 시점의 활성 예약은 낙관적 락으로 한 건만 성공해야 한다.
- 예약은 병원 승인형으로 운영한다.
- 병원의 수동 상태 변경은 스케줄러의 자동 전이보다 우선한다.
- 노쇼 슬롯은 반환하지 않는다.
- 동일 예약에 결제가 중복 승인되어서는 안 된다.
- 오프라인 정산 후 자동 재시도를 중단해야 한다.
- AI는 진단·처방·조회되지 않은 병원을 생성하지 않는다.
- AI 장애가 검색·예약·결제에 영향을 주지 않아야 한다.
- AI 구조화 출력 5필드는 모두 저장하고, 검색·집계용 2필드는 별도 컬럼에 중복 저장한다.
- **\[결정 필요 — 정본 충돌\]** 검색 Tool 실패 응답을 LLM이 생성할지, 서버가 `fallback=true`와 고정 문구로 직접 검색을 유도할지는 추후 결정하며 결정 전에는 구현하지 않는다.
상세 조건과 상태 전이는 프로젝트 정책 및 저장소 SA 정본을 따른다.
## 6. MVP 범위
### 포함
- 회원·반려동물 관리
- 특정 지자체 공공데이터 1회 시드 적재
- 제휴·비제휴 2계층 병원 검색
- AI 진료역량 추천과 병원 검색 Tool Calling
- 병원 승인형 예약과 진료 상태 관리
- 노쇼 판정·정정과 전 병원 통합 이력
- 빌링키 후불 결제와 오프라인 정산
- 알림 폴링
### 확장
- 공공데이터 주기 갱신 배치
- 병원 전용 프론트 화면
- 실시간 알림 push
- 환불·부분 환불·정정 후 재청구
- 필요 시 노쇼 자동 리스크 관리 재검토
