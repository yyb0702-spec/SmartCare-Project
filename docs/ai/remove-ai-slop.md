# AI 슬롭 제거

AI가 만든 티가 나는 저품질 코드·문서를 걸러내는 기준이다. 작성자는 PR 전에, 리뷰어는 검토 때 이 패턴을 확인한다.

## 슬롭 패턴

- 요구사항과 무관한 과한 추상화
- 의미 없는 helper·manager·util 남발
- 실제 검증 없이 "robust", "production-ready", "완벽한" 같은 주장
- TODO를 남기고 후속 설명 없음
- 실패를 숨기는 fallback (예외를 삼키고 기본값 반환)
- 문서와 코드가 서로 다른 이야기를 함
- 같은 내용을 여러 문서에 복붙 (정본 원칙 위반 — rule-source-map 참조)
- 실제 API 계약(`ApiResponse` 포맷·ErrorCode)과 다른 예시 응답

## 정리 규칙

- 불필요한 추상화는 제거한다.
- 검증하지 않은 주장은 제거하거나 미검증으로 표기한다 (판정 규칙은 [검증 가이드](../testing/verification-guide.md)).
- TODO는 담당자·이유·다음 단계를 적는다. 셋 중 하나라도 못 적으면 TODO 대신 질문 Issue로 분리한다.
- 중복 문서는 정본으로 모으고 나머지는 링크로 바꾼다 ([규칙 정본 지도](rule-source-map.md)).
- "완료" 표현은 PASS 근거가 있을 때만 사용한다.

## 문서 품질 기준

좋은 문서는 다음을 만족한다.

- 무엇을 할지 명확하다.
- 왜 그렇게 하는지 설명한다.
- 대안을 언급한다.
- 모르는 것은 모른다고 적는다.
- 읽는 사람이 다음 행동을 알 수 있다.
